CCT
Cognitive Control Training

A stream of numbers is spoken aloud. Apply the selected arithmetic operation to consecutive numbers and enter your answer before the next one arrives. Speed adapts to your performance.

Addition example: You hear 3, then 7 → type 10. Next you hear 2 → 7+2 → type 9.

Mode & Difficulty
Session mode

15-minute session, starting at 1500 ms. After every 5 trials: if 4 or more were correct → interval −100 ms (faster); otherwise → +100 ms (slower). Never drops below the minimum interval setting.

Target accuracy mode — algorithm gradually adjusts speed so your accuracy converges on a chosen percentage

Based on the 85% rule: research suggests learning is fastest when ~85% of responses are correct. The algorithm uses a rolling accuracy window and nudges the interval up or down to reach your target — slowly enough that it never feels abrupt. Overrides Smooth Adapt when active.

85%

Default 85% — the sweet spot for working-memory training. Lower = harder (faster pace); higher = easier (slower pace).

500 ms

The interval will never speed up past this floor. Default: 500 ms. Set to 0 to remove the limit.

Assist & Forgiveness
Number pool
1-digit mode — answers are always a single digit (0–9)

The number generator constrains each spoken number so that the computed answer never exceeds 9. Mainly useful for addition. For multiplication or division you may want to keep numbers small using the Number Range setting instead. Default: off.

Minimal input — only type the ones digit of the answer

E.g. if the answer is 13, typing 3 is accepted. If the answer is 9, typing 9 works as usual. Default: off.

Hide input — mask what you type so you can't see your own keystrokes

Removes visual interference from seeing your own input. The answer box stays visible and feedback (correct/wrong border colour) still shows — only the typed characters are hidden. Default: off.

Show equation stream — display a live running list of equations below the playing area

Shows each equation as it is answered, colour-coded green (correct) or red (wrong/missed). Newest at the top. Default: off.

Fumble correction
Fumble mode — accept digit-reversed inputs as correct (e.g. typing 41 when answer is 14)

Only triggers when the reversed number equals the correct answer and the typed value is not itself reachable.

Fat finger mode
Fat finger mode — if the correct answer is among multiple inputs entered at once, it still counts

E.g. if you fat-finger 154 when the answer is 14, it counts as correct. Limited to a maximum of 4 digits submitted at once — pushing all buttons won't give a free pass. Default: off.

Coyote time
Coyote time — a late-but-correct answer retroactively fixes the previous missed trial

Named after the video-game concept. If you knew the answer but typed it just after the timeout, the miss is reversed. The current trial is never affected.

500 ms

How far into the next trial a late answer still counts for the previous one.

ICT mode Inhibitory Control
ICT mode — inhibitory control training (go/no-go style)

When the expected answer and the previous expected answer together equal the theoretical maximum result for the current number range, that trial becomes a no-go stimulus. Instead of typing the answer, press ↑ Arrow Up to inhibit your response. Typing a number instead counts as a false alarm (wrong). Works best in addition mode. Default: off.

Silence only — only complete silence counts as a correct inhibition

When ticked, any input on a no-go trial — including ↑ Arrow Up — is a false alarm. The only correct response is to do absolutely nothing and let the trial time out.

Number Distinction Mode NDM
Number Distinction Mode (NDM) — press an arrow key to indicate whether the spoken number is smaller, equal, or larger than the previous one

After hearing the number, press the matching arrow key — based on how it compares to the previous spoken number (not the sum):

◀ Left — current number is smaller than the previous one ▶ Right — current number is larger than the previous one ▼ Down — current number is the same as the previous one

The arrow key can be pressed any time during the trial window alongside your number input. Correct number + wrong arrow = trial fails. The "previous number" follows the same n-back depth as the rest of the task. Cannot be active simultaneously with Sum Distinction Mode. Default: off.

Sum Distinction Mode SDM
Sum Distinction Mode (SDM) — press an arrow key to indicate whether the current sum is smaller, equal, or larger than the previous correct sum

After entering your number answer, press the matching arrow key — based on how the current sum compares to the previous correct sum:

◀ Left — current sum is smaller than the previous sum ▶ Right — current sum is larger than the previous sum ▼ Down — current sum equals the previous sum

The arrow can be pressed any time during the trial window. Correct number + wrong/missing arrow = trial fails. Compatible with ICT mode. The "previous sum" is the last correctly answered sum. Cannot be active simultaneously with Number Distinction Mode. Default: off.
